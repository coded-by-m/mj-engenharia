# Motion Direction

> Generated by Coded by M — Site Architecture System. Source of truth: `/docs/ai/DISCOVERY.md` and `/docs/ai/DECISIONS_LOG.md`.

## Core Motion Concept
"Calm precision." Motion confirms quality and guides the eye; it never performs. Everything feels engineered: short, eased, purposeful. Premium = restraint.

## Motion Language
- Easing: standard ease-out (`cubic-bezier(0.16, 1, 0.3, 1)` or similar) for entrances; quick ease for hovers.
- Duration: entrances 400–600ms; microinteractions 120–200ms.
- Distance: reveals translateY 12–16px + fade. No large slides or bounces.
- Stagger: list/grid items stagger 60–90ms for a controlled cascade.

## Hierarchy
1. **Primary narrative motion** — hero entrance (headline → sub → CTA, sequenced), section reveals on scroll.
2. **Section transitions** — fade + rise as sections enter viewport (IntersectionObserver, once).
3. **Microinteractions** — button/card hover elevation, link underline, WhatsApp button subtle attention.
4. **Decorative motion** — minimal; optional faint navy grid/parallax accent, performance-permitting.

## Page / Section Priorities
- **Hero:** sequenced reveal of H1, subhead, CTA, seal. Background image subtle scale-in (max ~1.03).
- **Especialidades:** the two flagship blocks rise in with light stagger; icon micro-pop on view.
- **Soluções / Diferenciais:** staggered card reveals; hover elevation + icon accent.
- **Quem Somos:** photo + text reveal in tandem.
- **Métricas (Quem Somos / proof):** count-up animation on first view (placeholders until real numbers).
- **CTA bands:** CTA button subtle, slow pulse/scale on hover; no constant looping that distracts.

## Microinteractions
- Buttons: hover lift 2px + shadow + red darken (`--red-800`); active compress.
- Cards: hover lift 2–4px, border/shadow strengthen.
- Links/nav: underline grow from left; active anchor highlight on scroll-spy.
- Form: focus ring transition; inline validation feedback.
- Floating WhatsApp: gentle entrance after scroll; optional single attention pulse, not infinite.

## Mobile Behavior
Lighter motion: keep fades, reduce/skip parallax and large background scaling; ensure scroll performance and battery friendliness; counters still animate but cheaply.

## Accessibility
- Respect `prefers-reduced-motion`: disable reveals/parallax/count-up, show final state instantly, keep only opacity if any.
- No motion conveys essential information.

## Performance Restrictions
- Do not compromise LCP, CLS or INP.
- Prefer CSS transitions/transforms + IntersectionObserver over JS animation loops.
- Avoid heavy animation libraries without need (small utility or hand-rolled only).
- No WebGL.
